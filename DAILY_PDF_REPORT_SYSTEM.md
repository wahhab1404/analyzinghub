# Daily PDF Report System

## Overview

The Daily PDF Report System automatically generates professional trading reports at market close (4 PM ET) and distributes them to all subscriber Telegram channels.

## Features

### 📊 Professional Report Design
- Beautiful gradient header with branding
- Comprehensive statistics grid showing:
  - Total trades
  - Active trades
  - Closed trades
  - Expired trades
  - Average profit %
  - Maximum profit %
  - Win rate %
- Detailed trade cards with:
  - Symbol and option type (CALL/PUT)
  - Strike price
  - Entry price
  - Highest price achieved
  - Current price
  - Max profit percentage
  - Trade status and exit reason

### 📅 Automated Scheduling
- Runs Monday-Friday at 4 PM ET (21:00 UTC)
- Only on trading days
- Automatically triggered via pg_cron

### 📱 Telegram Distribution
- Sends formatted reports to all analyzer channels
- Includes statistics and performance metrics
- Professional formatting with emojis and formatting
- Separate message for each subscribed channel

### 💾 Historical Tracking
- Stores reports in `daily_trade_reports` table
- Preserves HTML for future viewing
- Tracks daily statistics over time

## Architecture

### Components

1. **Edge Function**: `generate-daily-pdf-report`
   - Fetches today's trades
   - Calculates statistics
   - Generates HTML report
   - Stores in database
   - Sends to Telegram channels

2. **Cron Job**: `daily-pdf-report-generator`
   - Scheduled: Mon-Fri at 21:00 UTC (4 PM ET)
   - Triggers edge function automatically
   - Runs after market close

3. **Database Table**: `daily_trade_reports`
   - Stores daily report data
   - Preserves HTML content
   - Tracks metrics over time

## Statistics Calculated

1. **Total Trades**: All trades created today
2. **Active Trades**: Trades still open
3. **Closed Trades**: Trades closed today
4. **Expired Trades**: Trades that expired today
5. **Average Profit**: Mean max profit % of all trades
6. **Max Profit**: Highest profit % achieved
7. **Win Rate**: Percentage of profitable trades

## Usage

### Manual Testing
```bash
npm run test:pdf-report
```

This will:
- Generate today's report
- Send to all configured channels
- Store in database
- Display statistics

### Production Schedule
The report automatically generates at:
- **4:00 PM ET** (Eastern Time)
- **21:00 UTC** (Universal Time)
- **Monday through Friday**

### Viewing Reports

#### API Endpoint
```typescript
GET /api/indices/daily-reports?date=YYYY-MM-DD
```

#### Database Query
```sql
SELECT * FROM daily_trade_reports
WHERE report_date = CURRENT_DATE
ORDER BY created_at DESC
LIMIT 1;
```

## Configuration

### Environment Variables Required
- `SUPABASE_URL` - Auto-configured
- `SUPABASE_SERVICE_ROLE_KEY` - Auto-configured
- `TELEGRAM_BOT_TOKEN` - Required for Telegram delivery

### Channel Requirements
Channels must have:
- Valid `telegram_channel_id`
- Valid `telegram_channel_username`
- `is_active = true`

## Customization

### Styling
The HTML template uses inline CSS for maximum compatibility. Colors and styling can be customized in the `generatePDFHTML` function:

```typescript
// Header gradient
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);

// Stat card colors
.stat-card.primary { border-left-color: #667eea; }
.stat-card.success { border-left-color: #48bb78; }
.stat-card.warning { border-left-color: #ed8936; }
```

### Message Template
Telegram message format can be customized in `sendToTelegramChannels`:

```typescript
const message = `
📊 <b>Daily Trading Report</b>
...
`;
```

## Troubleshooting

### Report Not Generating
1. Check cron job status:
   ```sql
   SELECT * FROM cron.job WHERE jobname = 'daily-pdf-report-generator';
   ```

2. Check edge function logs in Supabase dashboard

3. Verify trades exist for today:
   ```sql
   SELECT COUNT(*) FROM index_trades
   WHERE created_at::date = CURRENT_DATE;
   ```

### Telegram Not Receiving
1. Verify bot token is set
2. Check channel configuration:
   ```sql
   SELECT * FROM analyzer_plans
   WHERE telegram_channel_id IS NOT NULL
   AND is_active = true;
   ```

3. Test bot permissions in channel

### Missing Statistics
- Ensure trades have required fields populated
- Check `max_profit_percent` calculations
- Verify `contract_high_since` is being tracked
